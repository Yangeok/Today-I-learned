# Kafka vs. Redis Pub/sub vs. RabbitMQ

- kafka
  - 용도
    - 대용량 처리, 실시간, 고성능, 고가용성이 필요한 경우 사용
    - 재처리가 반드시 필요한 경우 사용
  - 구성
- rabbitmq
  - 용도
    - 트래픽은 적지만 장시간 실행되고 안정적인 백그라운드 작업이 필요한 경우 사용
    - 정확한 요청/응답이 필요한 경우 사용
  - 구성
- redis pub/sub
  - 용도
    - consumer에 반드시 이벤트가 도착할 필요 없는 푸시알림이 중요한 경우 사용
  - 구성
